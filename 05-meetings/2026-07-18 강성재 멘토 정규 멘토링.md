# 0718 강성재 멘토 정규 멘토링 (백엔드 진행 보고)

> [!tldr]
> 백엔드 2주 진행 보고: 인프라 정비 → 영상 도메인 전체 구현 → 운영 안정화, 6덩어리.
> 핵심: dev/prod 분리 + CI/CD, 100×100m 격자 양자화·색칠 조회, presigned 업로드 파이프라인(FFmpeg 인코딩 포함), S3 무결성 보안 버그 수정, 관측 스택(Prometheus·Grafana)·k6, Swagger.
> 후반에는 디자인↔구현 정렬 문서 작업 (PRD·ERD 정렬·ADR 2건).

## 이 노트로 답할 수 있는 질문
- 0718 멘토링에서 뭘 보고했나?
- dev/prod 환경 분리는 어떻게 했나?
- 영상 업로드 파이프라인은 어떤 구성인가?
- S3 관련 보안 버그는 뭐였고 어떻게 막았나?
- 운영/모니터링에 뭘 구축했나?

## 개요
- 일시: 2026-07-18 · 멘토: 강성재 · 작성: 성민

## 내용
1. **개발 환경·CI/CD 분리** — dev/prod DB·Redis 컨테이너 격리(EC2 1대), GitHub Actions 분리 배포, CORS, As-Is/To-Be 다이어그램
2. **격자 시스템** — 100×100m 격자 양자화 유틸(좌표→격자 ID), 뷰포트 색칠 조회 API(커서 페이지네이션)
3. **영상 업로드 파이프라인** — presigned URL → S3 직접 업로드 → 메타 저장+점령 / FFmpeg 720p 인코딩 워커 + 썸네일 / 삭제 시 점령 롤백·같은 격자 교체
4. **S3 무결성** — 파일 없이 격자 점령 가능하던 보안 버그 수정(s3Key 소유권·실존·중복 3중 검증), 고아 객체 정리
5. **운영·모니터링** — Prometheus+Grafana+postgres_exporter, k6 부하 테스트, 배포 헬스체크, Flyway 파일 수정 차단 CI 가드(32시간 다운 사고 재발 방지)
6. **문서** — Swagger(JWT 포함), status.md 단일 진실, SA/IA/UseCase, API 명세 정리
