# 2026-07-23 기획회의 — 미션·이벤트 격자 표시 방식 (점·선·면)

> [!tldr]
> 녹음(25분) 전사 기반 회의록. 표시 방식 대부분 수렴: 축제=격자 미노출 **큰 면/원**(겹치면 투명도·색 구분), 코스=**직선 폴리라인 표현 + 격자 통과 판정**, 팝업=**핀/마커 + 클릭 시 정보 카드**(네이버 지도 음식점 UX).
> 미결: 미점령 격자 점선/실선(토글 비교하기로) · 핫구역 정의·디자인 · 코스 시작점/순번 표기.
> ⚠️ 모노 녹음이라 화자는 내용 기반 추정(A/B/C), 전사 오인식 보정 있음. 설계검토 문서와 배치되는 발언 2건(길찾기 CTA 불필요, 팝업=마커) — 확정 시 [[설계검토 미션·이벤트 기능 추가]] 갱신 필요.

## 이 노트로 답할 수 있는 질문
- 미션 유형별(축제·코스·팝업) 지도 표시 방식은 어떻게 수렴됐나?
- 축제가 겹치면 어떻게 표시하기로 했나?
- 코스를 직선 폴리라인으로 하되 판정은 어떻게 하나?
- 미점령 격자 점선 표시는 어떻게 하기로 했나? (미결 — 토글 비교)
- 이 회의에서 기존 설계검토 문서와 어긋난 발언은 뭔가?
- 액션 아이템과 담당(추정)은?

## 결정·수렴

1. **축제형 = 큰 면/원** — 81칸(9×9) 노출 시 "1칸만 찍은" 프레이밍 문제로 격자 미노출. 겹침은 공공데이터상 현재 없음, 생기면 투명도·색상 구분. 줌아웃 시 클러스터링으로 개수 표시 아이디어.
2. **코스형 = 직선 폴리라인 표현, 판정은 격자 통과 체크** — 격자 따라 그리면 계단식, 실제 보행 경로는 모름(트래킹 안 함). 백그라운드 경로 트래킹(그라운드플립 방식)은 회의적 → 보류.
3. **팝업(포인트형) = 핀/마커** — 주위에 있으면 지도에 마커, 누르면 네이버 지도 음식점처럼 정보 카드, 원거리 항목은 지도 이동. "(면보다) 차라리 마커" + 마커 디자인 필요.
4. **위치 기반 노출** — 반경 10~20km 내 미션 노출, 반경·정렬 기준은 기획이 정하면 프론트가 구현.
5. **미션 데이터 수급** — 축제=공공데이터(JSON), 두루누비=API 수급 후 파싱, 팝업=직접 수집[?]. 격자는 업로드 시 생성(기존 설계 재확인).

## 미결

- 미점령 격자 **점선/실선**: 지오비전(SKT)식 전체 점선은 "너무 과함" → 토글로 비교해보기로.
- **핫구역** 정의·범위·디자인: "업로드된 것 기준" 방향만. 미션 3종이 부가적으로 핫구역을 만드는 종합적 성격이라는 의견. [전사 저품질 구간]
- **코스 시작점·순번 표기**: "일단 격자 다 넣고 인덱싱 처리" 의견까지만.
- 바텀시트 **길찾기 버튼**: "필요 없다" 발언 — 설계검토 문서(격자 밖 CTA=길찾기)와 배치, 재확인 필요.
