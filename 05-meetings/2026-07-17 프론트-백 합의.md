# 프론트-백 합의 (지도 데이터 요청 방식)

> [!tldr]
> 지도 데이터는 bbox(뷰포트 좌표) 형태로 백엔드에 요청하기로 합의.
> 실시간 아님 — 지도의 새로고침 버튼을 누르면 API 요청하는 폴링 방식.
> 격자 영상 목록 응답 형태(GET /api/grids/{gridId}/videos, 커서 페이지네이션) 예시 합의.

## 이 노트로 답할 수 있는 질문
- 지도에서 격자 데이터를 어떻게 받아오기로 했나?
- bbox 요청 방식은 누가 언제 합의했나?
- 지도 갱신은 실시간인가 폴링인가?
- 격자 영상 목록 API 응답은 어떤 형태로 내려주기로 했나?

## 개요
- 일시: 2026-07-17
- 참석: KangJeong, 최규호
- 안건: 지도 데이터 수신 방식

## 내용
- 지도 데이터는 **bbox 형태**로 백엔드에 요청 (카카오맵 자체 제공 좌표 활용).
- 새로고침 버튼 → API 요청하는 **폴링 방식**.
- 격자 영상 목록 예시: `GET /api/grids/{gridId}/videos?sort=latest&size=20&cursor=...` — 응답에 gridId·regionName·videos(썸네일·상태·조회수)·nextCursor.
