> 멘토 피드백: usergrid를 Claude 룰로, FastAPI는 최악 케이스 실측하라(YOLO), 웹소켓은 커넥션 임계점을 명확히 정의하라, 모니터링은 장애 스토리로.

## 내용
- **기획 확정**: 서비스 정의(방문 장소를 30초 영상으로 기록, 100×100m 격자 수집), MVP=개인 도감만, glossary 단일 진실, SA(8개 서비스)·IA·User Journey.
- **인프라**: EC2 1대(t3.small)+Docker로 postgres-dev/prod(5432/5433)·redis-dev/prod(6379/6380) 볼륨까지 격리, fillmap.kr(Nginx+HTTPS), systemd fillmap-dev 상시 구동. RDS는 출시 때 전환.
- **CI/CD**: ci.yml(빌드+테스트, PostGIS 컨테이너 포함) / cd-dev.yml(develop→dev 자동배포) / cd-prod.yml(main→수동 승인 후). 첫 자동배포 성공 — 배포 경로 불일치(MSG-52)·SSH 포트 차단 해결.
- **멘토 피드백**: usergrid는 Claude 코드 룰로 지정 / FastAPI는 사람 많은 30초 영상 최악 케이스 실측(YOLO에 바로) / 폴링·웹소켓은 메인 화면 트래픽 임계점(커넥션 수)을 명확히 정의 / 모니터링은 "장애 인지→개선, 몇 초→몇 ms" 스토리로.
