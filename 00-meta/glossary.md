# glossary — 용어사전

> [!tldr]
> 저장소에서 쓰는 용어를 한곳에 정리. 새 용어가 생기면 여기에 추가.

## 이 노트로 답할 수 있는 질문
- 봉투(envelope)가 뭐야?
- canon이 뭔 뜻이야?
- ADR이 뭐야?
- compiled와 raw의 차이는?
- confluence-sync가 뭐야?

| 용어 | 뜻 |
| --- | --- |
| **봉투 (envelope)** | 모든 compiled 노트가 갖춰야 할 공통 구조 4요소 (frontmatter·tldr·질문·출처) |
| **canon** | 단일 진실. 한 곳에서만 정의, 변경은 사람 승인 필요 |
| **ADR** | Architecture Decision Record — 주요 결정의 스냅샷 기록 |
| **unfold** | 제품이 2개+ 될 때 flat 구조를 제품별 폴더로 확장하는 것 |
| **compiled** | raw를 정리한 .md 노트. git으로 버전관리 |
| **raw** | 날것 원본. Google Drive에 있고 불변 |
| **ingest** | raw → compiled 노트로 변환하는 과정 |
| **lint** | 위키 무결성 점검 (끊긴 링크·frontmatter 누락 등) |
| **confluence-sync** | 팀 Confluence(MSG)의 새/수정 페이지를 raw/confluence/ 스냅샷으로 가져오는 스킬. 데일리 스크럼·스프린트 회고 제외 |
