# 갭 분석 — 디자인 ↔ 문서·코드 싱크 (2026-07-17)

> [!tldr]
> "디자인이 정답" 전제로 Figma 시안 ↔ glossary/IA/SA/스키마/코드를 전수 대조. 해소 4건(42초 표기·전역노출 Phase2·서비스명 FillMap 확정·행정동 검색 설계), 블로커 2건(격자 표시명 → ADR로 해결 중, 담수율 대체 지표), 신규 이슈 1건.
> ⚠️ 신규 보안 이슈: 로컬 이메일 로그인이 프로파일 분기 없이 dev/prod에 열려 있음(@Profile 0건, 인증 메일도 없음) — 심사·시연 전 정리 필요.
> Apple 로그인 필요 확정(chk_users_provider에 APPLE 추가 + AppleOidcIdTokenVerifier). FE 공유 필수: 지도 홈 하단 시트(전역 집계)는 MVP에 구현 안 됨.

## 이 노트로 답할 수 있는 질문
- 시안과 코드가 어긋난 항목별 판정 결과는?
- "담수율 73%"는 뭐고 어떻게 처리됐나?
- 로컬 이메일 로그인의 보안 문제는 뭔가?
- Apple 로그인을 넣으려면 뭐가 필요한가?
- 서비스 이름 정본은 뭔가 (FillMap/FeelMap/MomentMap)?
- 검색바의 장소/격자/영상 검색은 각각 어떤 상태인가?
- 시안에 없지만 문서에 있는 것(스폰서·관리자·친구)은?

## 요약 (판정)
| 항목 | 판정 |
| --- | --- |
| 담수율 73% | 시안 오류 — 대체 지표 미정 🔴 |
| 격자 표시명 | 필요 확정 → [[ADR 격자 표시명 zone]] |
| 영상 42초 | 시안 오류 — 30초 유지 |
| 전역 노출(격자 24개·영상 138개) | MVP 밖(Phase 2 핫구역과 함께) — FE 공유 필수 |
| Apple 로그인 | 필요 — 스키마 CHECK + OIDC 검증기(Kakao 패턴 재사용) |
| 로컬 이메일 로그인 | 의도됨(로컬 전용)인데 프로파일 분리 안 됨 ⚠️ |
| 행정동 검색 | 설계 확정 — GET /api/regions/search |
| 서비스명 | FillMap 정본 (FeelMap 시안 수정, MomentMap 문서 정리) |

## 발견
- 검색 3종: 행정동 ✅ 자체 DB / 격자("홍대") 🔴 표시명 ADR에 묶임 / 영상 🔴 title 컬럼 자체가 없음. 최근 검색·방문 저장소도 없음.
- 프로필/설정 화면: 위치정보 동의·알림 설정 저장 테이블 없음, "수집률 서울 34%"는 집계 레벨 불일치(행정동 vs 시/도), 계정 삭제(탈퇴)가 시안에 없음(앱스토어 심사상 필수 가능성).
- SA/IA Actor 3종 중 시안은 사용자만 — 스폰서·운영자·친구는 MVP 범위 표기 필요.
- 잘 맞는 것: 지도 스타일(네이버 근거 뒷받침)·AI 블러·도감 3뷰·신고·스트릭·소셜 로그인 중심.

## 시사점 (후속)
즉시: 격자 표시명 설계(완료: ADR) · 담수율 대체 지표 · MVP 지도 홈 대안. 코드: Apple provider·로컬 로그인 프로파일 분리·regions 시딩·videos.title 검토. 문서: glossary/ia/architecture MVP 범위 표기, MomentMap 제목 정리.
