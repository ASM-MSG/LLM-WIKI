# IA v2 초안 — 화면·기능 트리 (EDGE 수준 drill-down)

> [!tldr]
> 기존 IA(에픽 8줄 수준)를 EDGE처럼 리프(화면 상태·액션) 단위까지 펼친 초안. 재료는 PRD 화면 7개 + 미션·이벤트 설계검토(v2) + 갭 분석.
> 각 리프에 [구현]/[부분]/[미구현]/[P2]/[결정필요] 태그를 달아 IA가 곧 개발 현황판이 되게 했다.
> 유스케이스도 "에픽 단위" 피드백을 반영해 사용자 목표(user goal) 단위 분해안을 함께 담았다. 팀 리뷰 후 drawio로 옮긴다.

## 이 노트로 답할 수 있는 질문
- FillMap의 화면·기능 전체 트리는 어떻게 생겼나?
- 미션·이벤트(축제·코스·스탬프북)는 IA 어디에 들어가나?
- 유스케이스를 어느 단위까지 쪼개야 하나? (에픽 → 사용자 목표)
- 어떤 리프가 구현/미구현/결정필요 상태인가?

## 0. 원칙

- **리프 = 사용자가 화면에서 하는 액션 또는 보는 상태.** EDGE IA의 drill-down 수준을 기준으로 한다.
- **리프마다 상태 태그**: `[구현]` `[부분]` `[미구현]` `[P2]`(Phase 2) `[결정필요]`(PRD §4·미션 문서 §12의 미결 항목).
- 미점령 격자 미표시 정책 때문에 첫 화면이 비어 보이는 문제 → 미션 레이어가 안내판 역할 (설계검토 §1).

## 1. IA 트리

```text

│   ├─ 스플래시·온보딩 (서비스 소개, 위치·카메라 권한 요청)
│   └─ 로그인
│       ├─ 카카오 소셜 로그인 (OIDC)                    [구현]

│   ├─ 지도 뷰
│   │   ├─ 내 점령 격자 색칠 (뷰포트 조회)               [구현]
│   │   ├─ 미션 레이어 (유형별 표현 — 판정은 전부 격자)

│   │   │   ├─ 코스: 폴리라인 (PATH)                     [미구현]

│   │   │   └─ 구역: 행정동 경계 (REGION)                [미구현]
│   │   └─ 지도 모드 토글 (나 / 나+친구)                 [P2]

│   │   ├─ 🥾 추천코스 N                                 [미구현]
│   │   ├─ 🔥 핫구역                                     [미구현·결정필요(MVP 여부)]
│   │   └─ fallback: 내 동네 채우기 (구역형)             [미구현]
│   ├─ 미션 바텀시트 (지도를 떠나지 않는 3단 시트)
│   │   ├─ half: 미션 리스트 (기간·거리·참여 수)          [미구현]
│   │   ├─ 항목 탭 → fitBounds 이동 + 영역 하이라이트 + peek
│   │   ├─ full: 내용·기간·홈페이지
│   │   ├─ 진행도: target=1 → "다녀옴 ✓" / target≥2 → "166칸 중 17칸"
│   │   └─ 거리별 CTA: 격자 안 "📹 여기서 찍기" / 밖 "🧭 길찾기"
│   └─ 요약 카드
│       ├─ 핫구역 TOP 5                                  [미구현·결정필요]
│       ├─ 미방문 격자 추천                              [미구현·결정필요(로직)]
│       ├─ 내 수집 현황 (격자 수·수집률·스트릭)           [미구현]
│       └─ 지역별 수집률 (구 단위)                        [미구현]
│
├─ ② 격자 썸네일 뷰 (격자 탭 시)
│   ├─ 영상 썸네일 그리드                                 [미구현(목록 API)]
│   ├─ 격자 요약: ID·총 영상 수·최근 업로드               [부분]
│   └─ 핫구역 지수·대표 태그·누적 조회수                  [미구현]
│
├─ ③ 핫구역 뷰
│   ├─ 실시간 순위 리스트 (최근 10분 업로드 기준)          [미구현·결정필요]
│   ├─ 갱신 시각 · 다음 갱신 카운트다운
│   └─ 항목 탭 → ④ 격자 상세

├─ ④ 격자 상세 보기
│   ├─ 격자 표시명 ("홍대입구 A-14")                      [미구현·결정필요(ADR)]
│   ├─ 행정 지역명 · 지역 수집률                          [미구현]
│   ├─ 격자 내 영상 목록 · 재생 (presigned GET)           [미구현]
│   ├─ 활동 지표 (수집률·핫구역 지수·총 영상)             [미구현]
│   ├─ 인근 미방문 격자 추천                              [미구현·결정필요]
│   └─ 영상 신고 → ⑤ 신고 모달                           [미구현]
│
├─ ⑤ 영상 신고 모달
│   └─ 신고 사유 선택 → 제출                              [미구현·결정필요(사유 enum·처리 플로우)]
│
├─ ⑥ 영상 촬영·업로드
│   ├─ 촬영 (격자 내 GPS 검증) / 갤러리 선택              [구현(파이프라인)]
│   ├─ 자동 압축·인코딩 + 진행률                          [구현(폴링 API 확인 필요)]
│   ├─ AI 하이라이트 추천 → 구간 조정                     [미구현·결정필요(MVP 여부)]
│   ├─ AI 자동 블러 확인 → 토글                           [미구현·결정필요]
│   ├─ 완료 → 점령/재방문 판정 → 격자 썸네일 반영          [구현]
│   ├─ (미션 기간·격자 일치 시) 스탬프 획득 풀스크린        [미구현]
│   └─ 내 영상 교체 / 삭제 (점령 롤백)                     [구현]
│
├─ ⑦ 개인 도감
│   ├─ 요약 (총 격자·방문 지역·업로드 영상·현재 스트릭)    [미구현]
│   ├─ 3뷰: 지도 / 갤러리 / 뱃지                          [미구현]

│   ├─ 스트릭·뱃지 성취                                   [미구현]
│   ├─ 스탬프북 (미션 — "부산 축제 15개 중 3개")           [미구현·결정필요(MVP/P2)]
│   └─ 공개 범위 설정 (MVP: 공개/비공개 · 친구공개는 P2)   [미구현·결정필요]
│
├─ 설정·프로필
│   ├─ 프로필 (닉네임·격자 색상)                          [미구현]
│   ├─ 로그아웃                                          [구현]
│   ├─ 계정 삭제                                          [미구현]
│   └─ 알림 설정                                          [P2]
│
├─ 소셜                                                   [P2 전체]
│   ├─ 친구 요청·수락·목록·차단
│   ├─ 친구 도감 보기
│   └─ 좋아요
│
├─ 운영자 (Admin Console)
│   ├─ 신고 처리 · 사용자 차단                            [미구현]
│   ├─ Trust Score · 통계 모니터링                        [미구현]
│   └─ 미션 수동 등록 (데모 팝업 1~2건)                    [미구현]
│
└─ 광고주·기업 (Sponsor Portal)                           [P2 전체]
    ├─ 캠페인·스폰서 격자 관리
    └─ 성과 리포트 조회
```

## 2. 유스케이스 세분화 — 에픽 → 사용자 목표 단위

멘토 피드백(2026-07-21): **"에픽은 유스케이스가 될 수 없다"** — 유스케이스의 알갱이는 EDGE IA의 마지막 리프 수준이어야 한다. 기준을 이렇게 확정한다:

> **유스케이스 = §1 IA 트리의 리프 중 액터가 시스템에 하는 행동, 1:1 대응.**
> 에픽은 유스케이스가 아니라 그룹(패키지) 박스로만 남긴다. 보조 흐름은 «include»/«extend»로 연결한다.
> 단, IA 리프 중 **표시 전용·시스템 규칙**(미점령 미표시, 정렬 고정, 상태 머신 등)은 액터의 행동이 아니므로 유스케이스로 올리지 않는다 — 유스케이스가 버튼 목록이 되는 것을 막는 경계선.

이 기준으로 뽑은 **유스케이스 57개**가 정본이며 다이어그램(`0_FillMap_UseCase_v2_draft`)에 반영돼 있다. 아래 표는 에픽 → 리프 대응의 요약이다.

| 기존 에픽 (현행 유스케이스) | 분해안 (목표 단위 리프) |
| --- | --- |
| 5초 촬영·업로드 | 현재 위치에서 영상 촬영 · 갤러리에서 선택 · AI 하이라이트 구간 확인/조정 «extend» · AI 블러 확인/토글 «extend» · 업로드 진행 확인 · 영상 교체 · 영상 삭제 |
| 지도 탐색·핫구역 | 내 점령 격자 확인 · 격자 영상 목록 열람 · 격자 상세(표시명·지표) 조회 · 핫구역 순위 조회 · 미방문 격자 추천 받기 |
| 미션·이벤트 (신규) | 주변 미션 칩으로 탐색 · 축제/팝업 상세 확인 · 코스 진행도 확인 · 미션 격자에서 촬영해 스탬프 획득 · 스탬프북 조회 · 길찾기로 이동 |
| 개인 도감·게임화 | 수집 요약 조회 · 지역별 수집률 조회 · 뱃지 목록 확인 · 스트릭 확인 · 도감 공개범위 변경 |
| 계정·권한 | 카카오 로그인 «include» 위치권한 · 로그아웃 · 계정 삭제 · 프로필(색상) 변경 |
| 안전 | 영상 신고 제출 · (운영자) 신고 처리 «extend» 사용자 차단 |
| 운영 | 통계 모니터링 · Trust Score 관리 · 미션 수동 등록 (신규) |
| 스폰서 활동 (P2) | 캠페인 관리 · 스폰서 격자 지정 · 성과 리포트 조회 |

액터는 현행 3종(사용자·운영자·스폰서) 유지. 미션은 별도 액터가 아니라 사용자 에픽으로 추가.
